# Submission Checklist

> OMH-BSE runs through this on April 26 before submission.

## Repo Quality
- [ ] README.md has: project description, features, architecture diagram, setup instructions
- [ ] `.gitignore` excludes: `.env`, `__pycache__`, `node_modules`, `.idea`
- [ ] No secrets committed (search for API keys, tokens)
- [ ] All debug/print statements removed
- [ ] License file present (Apache 2.0 to match OM)

## Setup Verification (Clone → Run)
- [ ] Fresh clone on clean machine
- [ ] `pip install -e .` works
- [ ] Docker OM starts without errors
- [ ] Bot JWT token generation documented
- [ ] Set env vars → run → chat works in <5 minutes

## Demo
- [ ] Demo video recorded and uploaded
- [ ] Video shows all 3 core features
- [ ] Video length: 2–3 minutes
- [ ] Video resolution: 1920×1080
- [ ] Clear audio or captions

## Hackathon Form
- [ ] Team name and members listed
- [ ] Correct paradox selected (T-01)
- [ ] GitHub repo URL correct
- [ ] Demo video URL included
- [ ] Related GitHub issues linked (#26645, #26608)

## GFI Track
- [ ] At least 1 GFI PR raised
- [ ] PR follows upstream contribution guidelines
- [ ] PR has screen recording (for UI fixes)

## Post-Submission
- [ ] Star the repo ⭐
- [ ] Comment on claimed issues with submission link
- [ ] Share in OM Slack community
